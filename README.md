# 通信会社の顧客離反（チャーン）予測

Kaggleの公開データ（Telco Customer Churn）を用いて、通信会社の顧客の解約を予測し、
「誰に・なぜ・どんな施策を打つべきか」まで導いた分析プロジェクト。
単なる予測に留めず、EDA・複数モデル比較・不均衡対策・SHAPによる要因分析・施策提案までを一気通貫で実施した。

##  プロジェクト概要

- **課題**: 通信会社の顧客（約7,000件）から、解約しそうな顧客を予測し離反を防ぐ
- **データ**: [Telco Customer Churn](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)（Kaggle）
- **成果物**: [分析ノートブック](./telco-churn-prediction.ipynb)
- **実行環境**: AWS SageMaker Studio

##  使用技術

- Python / pandas / numpy
- scikit-learn（ロジスティック回帰・決定木・SVM・KNN）
- XGBoost / LightGBM / CatBoost
- SHAP（要因分析）/ matplotlib / seaborn
- AWS SageMaker（モデル学習・デプロイ）/ Amazon S3 / boto3

##  分析の流れ

1. **EDA**: データの素性を把握し、解約と関係しそうな特徴量の仮説を立てる
2. **前処理**: 型崩れ（TotalChargesの空白混入）の修正、ダミー化、層化分割
3. **モデル比較**: 7モデルを比較し、最適なモデルを選定
4. **不均衡対策**: class_weight・閾値調整で少数派（解約）の検出力を改善
5. **要因分析**: SHAPで「何が解約に効くか」を可視化し、施策提案へ

##  主な発見と結論

- **モデル選定**: 表形式かつ線形寄りのデータのため、複雑な勾配ブースティングよりも
  シンプルな**ロジスティック回帰が最良**（「複雑なモデルが常に優れるとは限らない」ことを確認）
- **評価設計**: クラス不均衡（解約は約27%）のため、accuracyではなく
  解約クラスの recall / precision / f1 を主指標に設定
- **不均衡対策**: 閾値を0.274に調整し、解約recallを 0.56 → 0.75 に改善（取りこぼしを削減）
- **要因分析**: 継続月数の短さ・月契約・高い月額料金・光ファイバー利用が主な解約要因（EDA仮説と一致）

##  施策提案

分析結果から、以下の施策を提案：

1. **契約初期の顧客を手厚くフォロー**（継続月数の短さが最大の解約要因）
2. **長期契約への誘導**（月契約層に割引付きで長期契約を促す）
3. **高額・光ファイバー層のケア**（料金プラン見直し・品質サポート強化）
4. **オプション加入の促進**（セキュリティ/サポート加入で継続率向上）

閾値0.274で解約可能性の高い顧客を抽出（解約者の約75%を捕捉）し、上記施策を優先投下する。

##  モデルのデプロイ（発展）

分析で作成したモデルを、AWS SageMakerでデプロイし、「APIで解約予測が返る」状態まで実装した。

- 学習済みモデルをアーティファクト化してS3に保存し、**リアルタイム推論エンドポイントとしてデプロイ**
- 顧客データを送ると、解約確率＋施策判定（閾値0.274で解約リスク有無を判定）が返るAPIを構築
- 推論時の500エラーに対し、**CloudWatchログから原因（送信データのカラム名欠落）を特定 → 推論スクリプトを修正**して解決
- エンドポイントの作成から削除・コスト管理まで一連を実施

→ 「モデルを作って終わり」ではなく、本番で使える形にするまでを経験した。
